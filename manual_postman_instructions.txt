### üîç Individual test cases (executed in Postman)

**Precondition:** both the proxy (`http://localhost:8000`) and downstream
(`http://localhost:8085`) services are running and healthy.

1. **Downstream ‚Äì missing `user` field**
   - Given a POST to `http://localhost:8085/login` with the `missingUser`
     payload.
   - When the downstream service processes the body.
   - Then it should return **400** with a message like
     `"Missing 'user' key"`.

2. **Downstream ‚Äì valid request**
   - Given a POST to `http://localhost:8085/login` carrying
     `validUser`.
   - When the service handles the request.
   - Then the response should be **200**, echoing `user: "bob"`, plus a
     `token` and `timestamp` in the JSON body.

3. **Proxy ‚Äì request without user**
   - Given a POST to `http://localhost:8000/login` using `missingUser`.
   - When the proxy inspects the request.
   - Then it should immediately reply **400** without consulting downstream.

4. **Proxy ‚Äì normal forwarding**
   - Given a POST to `http://localhost:8000/login` with `validUser`.
   - When the proxy forwards the request to downstream.
   - Then the proxy should return **200**, strip the `user` field from the
     downstream JSON, and preserve other fields like `token` and `timestamp`.

5. **Proxy ‚Äì downstream omits user**
   - Given a POST to `http://localhost:8000/login-no-user` with `validUser`.
   - When the downstream endpoint deliberately drops `user`.
   - Then the proxy should respond with **400**, treating the missing value as an
     error.

6. **Proxy ‚Äì downstream returns non‚ÄëJSON**
   - Given a POST to `http://localhost:8000/login-invalid` with `validUser`.
   - When the proxy attempts to parse the downstream reply as JSON.
   - Then parsing fails and the proxy surfaces a **500** error.

7. **Proxy ‚Äì client sends invalid JSON**
   - Given a POST to `http://localhost:8000/login` with `Content-Type:`
     `text/plain` and the raw text `this is not json`.
   - When the proxy tries to consume the body.
   - Then it rejects the request with a **400** before hitting downstream.
